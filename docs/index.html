HPC Research Project

Task-Based Parallel Runtime with Work-Stealing Scheduler

A compact C++ runtime for DAG-based execution, dynamic load balancing, and benchmark-driven analysis.

View Repository

Project Overview

The system executes tasks as a directed acyclic graph on a fixed-size thread pool. Each worker owns a local deque, and idle workers steal tasks to maintain utilization under irregular workloads.

System Architecture

Task Graph

Tasks store a callable body, successor list, and remaining dependency counter.

Scheduler

Workers pop local work in LIFO order and steal remote work in FIFO order for balance.

Evaluation

Benchmarks report execution time, speedup, and efficiency for 1, 2, 4, and 8 threads.

Benchmark Results

Key Insights

Work stealing improves balance when the DAG exposes uneven ready work, but efficiency drops as synchronization and memory traffic become more visible at higher thread counts.
